Remote A Overview

This is an independent microfrontend that can be developed, tested, and deployed separately from the host application and other remotes.

Capabilities

  • Independent versioning
  • Separate deployment pipeline
  • Own routing configuration
  • Isolated state management
  • Team autonomy

Integration

This remote is dynamically loaded by the host application using Dynamic Module Federation. The integration is seamless and transparent to the end user.
